# ADR-0002：使用分布式同步选择

- 状态：Accepted
- 日期：2026-06-22
- 决策者：ARCH-ORCH

## 决策

主节点同步统一 release 和 `FLEET_PROXY` 选择；每台子节点直接连接真实代理节点。未命中规则由子节点本机 `DIRECT` 出口发送。

## 理由

- 主节点不承担全部带宽和额外一跳；
- 主节点故障不影响既有子节点数据面；
- 未命中流量可以保留子节点自身出口 IP；
- 统一节点切换可通过本机 Mihomo API 实现。

## 代价

- 每台子节点必须持有节点凭据；
- 节点切换为最终一致或分阶段一致；
- 真实节点列表必须同步一致并使用稳定 node_id。

## 排除方案

中央 Gateway 不作为 V1，但接口保留未来 egress_mode 扩展空间。
